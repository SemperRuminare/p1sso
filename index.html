The transition to Platform One Single Sign-On (P1 SSO) has begun. Scroll down for more information and setup instructions.

What is Platform One SSO?

Platform One Single Sign-On (P1 SSO) allows us to provide access to different apps using the same login based on an open source identity service called Keycloak. In other words, you will have one login to access apps such as Mattermost, Jitsi, GitLab, etc. It also gives us the capability to have multiple login methods on the same account, meaning you'll be able to use a username, password and MFA to log in at home while using a CAC to log in at work. These changes will be rolling out in two phases:

Phase 1: Migrate Mattermost username/password/MFA users to P1 SSO. This has already begun and will be complete on 31 May.

Phase 2: Migrate Mattermost AF Portal Login users to P1 SSO and enable the CAC login option. This will follow shortly after Phase 1 (approximately end of June).

It's important to note that P1 SSO currently only supports the username/password/MFA login option. We're hoping to enable the CAC login option soon and will make an announcement when it's available..

What do I need to do?

AF Portal Login Users: If you currently log into Mattermost using the AF Portal Login button, do not do anything right now. Continue logging into Mattermost using the AF Portal Login button.

Username/Password/MFA Users: If you log into Mattermost using a username, password and MFA, you can set up your P1 SSO account now using the instructions below. If you don’t set up your P1 SSO account before 31 May, you will simply be required to set up your P1 SSO account in order to access Mattermost. This process does require access to your work email which is why we recommend you setup your P1 SSO account when it's convenient for you.

How do I set it up?

Access to your mobile device MFA app (e.g. Google Authenticator, Authy, etc.) and the email address associated with your Mattermost account is required. It's also recommended to perform these steps from a web browser on a computer.

During this process, you will be setting up your P1 SSO account which is separate from your current Mattermost account. In other words, you will have a username, password and MFA 6-digit pin for P1 SSO that is separate from your username, password and MFA 6-digit pin for Mattermost. You will still need your Mattermost credentials even after you create your P1 SSO account, so be sure not to delete them!


Step 1

Go to https://sso.collab.cdl.af.mil/ and select “Forgot Password?”. Enter the email address associated with your Mattermost account and click “Submit”.


Step 2

You will receive an email with a link to reset your credentials. Click “Link to reset credentials” or copy the URL into a web browser.


Step 3

Setup MFA by scanning the QR code with your MFA app. This will create a new MFA 6-digit pin which you'll use for P1 SSO. Enter the new 6-digit pin and type in a device name (can be anything). You will still need your previous MFA 6-digit pin to log into Mattermost prior to 31 May.

Note: If your MFA pin is not accepted or entered incorrectly, the QR code is reset so you will need to scan it again.


Step 4

Enter a new password. You can use the same password as your Mattermost account but it must contain at least two special characters.


Optional

Feel free to update the rest of your profile but do not change your email address or username.


That's it for now!

Continue using Mattermost as you have been, meaning the way you log into Mattermost has remain unchanged. Continue using your existing Mattermost username, password and MFA 6-digit pin. After 31 May, we will transition to P1 SSO which will require you to log into Mattermost by selecting “Platform One SSO” from the Mattermost login page. This is when you will start using the username (should be your same email address), password and MFA 6-digit pin you just created following these steps.

We understand it's a bit confusing having two separate sets of credentials, especially having two MFA 6-digit pins. This is a major but necessary transition to be able to bring much requested features and apps in the near future.


Don't want to wait?

If you don't want to wait until 31 May for the transition, you can manually switch your Mattermost sign-on method now to start using P1 SSO. Again, it's important to note you will still only be able to log in using a username, password and MFA for now. The ability to also log in with a CAC to P1 SSO is a feature we're hoping to have avaialble by the end of June. If you want to change your Mattermost sign-on method now, click here for instructions.

FAQ

Why is this change necessary?

  • Platform One Single Sign-On (P1 SSO) gives us the ability to provide access to different apps such as Mattermost, Jitsi, GitLab, etc. using one login. This prevents having to create a new account for each app or service we release. Additionally, using P1 SSO will allow users to have multiple login methods such as username, password and MFA at home and using a CAC to login at work. Some of these changes were also made to help improve security.

    • Why do I have to set up a new password and MFA again?

  • Unfortunately, we are not able to migrate passwords from Mattermost. However, you can use the same password you were using in Mattermost as long as it contains two special characters. Since the P1 SSO is a different login service, MFA must be setup again.

    • Why am I not receiving a password reset email from P1 SSO?

  • Try having the email sent again and make sure to check your junk email folder.
  • If you created your Mattermost account after 19 May, your P1 SSO account may not have been automatically created. Ask a team or system admin for an invite link to create a P1 SSO account.

    • What username, password and MFA pin do I use to log into Mattermost?

  • Before 31 May, continue using your existing username, password and MFA pin to access Mattermost (i.e. what you have been doing).
  • After 31 May, you will log in using the P1 SSO sign-on method which will use your email address and the newly created password & MFA pin you set up in the steps above.

    • How does a new user create a Mattermost account with a username, password and MFA?

  • New Mattermost accounts using username, password and MFA are now created through P1 SSO. To help increase our security posture, they will need an invite link from another user or admin.
  • Users still have the option to create new Mattermost accounts using the AF Portal Login option, but they will be limited to only CAC access. Ultimately, when both phases of the P1 SSO transition is complete, all users will use a P1 SSO account to access apps like Mattermost and will be able to log in using username, password and MFA or a CAC.

    • How do I send someone an invite link to P1 SSO?

  • Administrators and users can find invitations links on their https://sso.collab.cdl.af.mil/ profile. Invitation links are valid for 10 days. There is no restriction on sharing invitation links as long as it is done through a government approved medium.

    • I see a Platform One SSO login button on the Mattermost login screen. What's it for?

  • The functionality already exists to start using your P1 SSO to access Mattermost but it requires you to change your Mattermost sign-on method. While it is possible to do this individually, accounts will automatically be switched over on 31 May.
